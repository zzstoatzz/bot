# memory

phi has three memory systems. they differ in visibility, trust level, and who curates them.

## 1. thread context (chronological)

**source**: ATProto network
**storage**: none — fetched on demand
**visibility**: public (it's posts)

```
@alice: I love birds
@phi: me too! what's your favorite?
@alice: especially crows
```

fetched via `client.get_thread(uri, depth=100)` when processing a mention. provides what was said in THIS thread. not cached — the network is always current (~200ms fetch).

## 2. private memory (TurboPuffer)

**source**: phi's own extraction and tools
**storage**: TurboPuffer vector DB (OpenAI text-embedding-3-small embeddings)
**visibility**: private to phi

### namespaces

| namespace | contents |
|-----------|----------|
| `phi-core` | identity and guidelines (highest trust) |
| `phi-users-{handle}` | per-user observations + raw interaction logs |
| `phi-episodic` | phi's own notes about the world |

each user gets an isolated namespace. within a user namespace, rows have a `kind`:
- `observation` — extracted facts about the user ("likes rust", "name is nate")
- `interaction` — verbatim log of what was said ("user: X / bot: Y")
- `summary` — compacted relationship summary (generated by external prefect flow)

### schema

observations carry two fields for lifecycle management:
- `status` — `"active"` or `"superseded"`. only active observations appear in context injection and reconciliation. rows without `status` (pre-migration) are treated as active.
- `supersedes` — id of the observation this one replaced. empty string if original. forms a provenance chain: you can trace how an observation evolved.

### the extraction pipeline

after every conversation, `after_interaction()` runs two steps:

```
mention comes in
  → phi responds
  → store_interaction(): save verbatim exchange
  → extract_and_store(): run extraction pipeline
```

**extraction** (`extraction.py`):
1. send the new exchange to claude haiku — no existing observations in the prompt
2. haiku returns `list[Observation]`, each with `content` + `tags` (0-3, enforced by pydantic)

extraction runs blind — it only sees the current exchange. this prevents the feedback loop where the extraction model pattern-matches off existing (potentially bad) observations and reproduces their errors.

**reconciliation** — for each extracted observation:
1. find the 3 most similar active observations (vector search, excludes superseded)
2. send to haiku: "NEW vs EXISTING — ADD, UPDATE, DELETE, or NOOP?"
3. execute the decision:
   - **ADD**: write new observation with `status: "active"`
   - **UPDATE**: mark old observation `status: "superseded"`, write merged version with `supersedes: <old_id>`
   - **DELETE**: mark old observation `status: "superseded"`, write new version with `supersedes: <old_id>`
   - **NOOP**: discard the new observation

reconciliation is append-only — old observations are never deleted from turbopuffer. they're marked superseded so they stop appearing in context but remain as provenance. you can always trace what phi believed and when it changed.

### what curates this today

- **reconciliation on ingest**: ADD/UPDATE/DELETE/NOOP per observation, runs after every exchange (append-only — supersedes rather than deletes)
- **relationship summaries**: external prefect flow compacts observations into prose summaries (doesn't touch the source observations)

### what this sets up

- **phi-driven curation**: phi can review its own observations and supersede stale ones without losing history
- **promotion pipeline**: stable observations can be promoted to semble collections when there's enough data
- **retrieval stats**: can track which observations are actually useful by logging retrieval hits against evidence ids

## 3. public memory (cosmik/semble)

**source**: phi's tools (`save_url`, `note`, `create_connection`)
**storage**: phi's PDS (ATProto records), indexed by semble
**visibility**: public to everyone

phi writes three record types:
- `network.cosmik.card` (NOTE) — text notes / thoughts
- `network.cosmik.card` (URL) — bookmarks with title/description
- `network.cosmik.connection` — semantic links between entities (URLs or AT-URIs)

these are indexed by [semble](https://semble.so) and searchable by anyone via `search_network`.

### bookmarks

`save_url()` writes only to PDS (cosmik card). phi finds its own bookmarks via `search_network`. the `note()` tool still dual-writes to turbopuffer episodic — that's intentional private memory, not a bookmark.

## context injection

when phi processes a mention from `@alice` about topic X:

```
[CORE IDENTITY AND GUIDELINES]        ← phi-core namespace (highest trust)
[PHI'S SYNTHESIZED IMPRESSION]        ← relationship summary (lowest trust)
[OBSERVATIONS ABOUT @alice]           ← user namespace, kind=observation, status!=superseded
[PAST EXCHANGES WITH @alice]          ← user namespace, kind=interaction
[PHI'S RELEVANT MEMORIES]             ← episodic namespace, semantic search
[CURRENT THREAD]                      ← ATProto network fetch
[TODAY]: 2026-04-05                   ← date
```

each section is labeled with its trust level. phi's operational instructions tell it to trust current user messages over stored observations, and to flag synthesized impressions as unreliable.

## the graph (`/memory`)

the memory graph visualization shows phi + user nodes positioned by semantic similarity of their observation vectors. it's a social graph: phi at center, users around it, positioned by how similar their observation embeddings are (PCA projection). only active observations contribute to positioning — superseded ones are excluded.

## two curation surfaces

phi's knowledge lives in two places with different properties:

| | TurboPuffer (private) | cosmik/semble (public) |
|---|---|---|
| **who writes** | extraction agent (automatic) + phi's tools | phi's tools (intentional) |
| **who reads** | phi (via context injection + recall) | anyone (via semble search) |
| **who curates** | reconciliation agent (append-only supersession) | nobody yet |
| **trust** | medium — extraction can misattribute | higher — phi chose to write these |
| **growth** | bounded by supersession (old rows hidden, not deleted) | bounded by phi's intentional actions |

## summary

| | thread context | private memory | public memory |
|---|---|---|---|
| **what** | messages in current thread | patterns across conversations | knowledge worth sharing |
| **when** | this conversation | all time | all time |
| **how** | chronological | semantic similarity | semantic search (semble) |
| **storage** | network (ATProto) | TurboPuffer | PDS (cosmik) + semble |
| **visibility** | public | private to phi | public |
| **curation** | network handles it | extraction + append-only reconciliation | phi's intentional writes |
| **trust** | high (verbatim) | medium (extracted) | higher (intentional) |
